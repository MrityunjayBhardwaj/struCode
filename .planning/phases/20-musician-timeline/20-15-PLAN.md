# Phase 20-15 — Strudel.cc parity hardening

## GOAL (goal-backward anchor)

Real-world Bakery parity moves materially up from the 2026-05-15 baseline
(4/10 live patterns structurally parsed) by closing 6 named, reproducible
parser-gap classes in `packages/editor/src/ir/parseStrudel.ts`, while:

1. The 6 known repros become permanent CI fixtures and ≥9/10 of the known
   set parses structurally (D-04).
2. The 16 canonical corpus snapshots + 1551 editor tests stay byte-for-byte
   unchanged, EXCEPT explained structural improvements (e.g. arpoon #132).
3. A reusable `skipWhitespaceAndLineComments` primitive + label→trackId
   wiring + binding map are left as inheritable substrate.
4. The parser remains a STRUCTURAL MATCHER, not a JS interpreter (the
   sacred line; D-02/D-04 enforce it).

Every wave's tasks below chain to this GOAL; the verification wave measures
it directly.

## RESOLVED PLANNER DECISIONS (research open-questions answered)

These bind the tasks below. They are resolved here so executors do not
re-litigate:

  `stripParserPrelude`'s whole-line skip (pS:139-148) to the shared
  primitive. It is a line-classifier, structurally distinct from
  inter-token skip, and reshaping it risks the multi-line boot-call depth
  logic (pS:161-212). The shared primitive serves the 3 genuinely
  inter-token sites: `applyChain` (reference), `splitArgsWithOffsets`
  (#137/G4), `extractTracks` label scan (#138/G5). PV49 is updated in the
  catalogue to document this divergence (α has a dedicated catalogue task).
- **R2 (#135 misread):** issue #135's "reuse the α-6 settingPatterns audit
  as authoritative skip-set source" is a MISREAD — α-6 covers theme/font
  UI setters (`{theme, fontFamily, fontSize}`), which are CHAIN methods,
  not top-level boot calls, and not tempo setters. The real anti-drift
  mechanism is: a code comment at pS:125-128 citing the upstream
  `@strudel/core` file + pinned Codeberg SHA `f73b3956`, plus one CI
  fixture per added setter. There is NO programmatic cross-ref (upstream
  list is not vendored) — document this explicitly. Setter family to add:
  `setcpm` (proven by repro), and any pure no-return side-effect setter
  found in the upstream `@strudel/core` index export audit (task α-1
  enumerates the exact final list after the audit).
- **R3 (#132 s-vs-note semantics):** the inner bare string in

  Recursing naively through `parseExpression` loses this. RESOLVED via a
  mandatory Lokāyata probe (task β-1) run BEFORE the impl task (β-2). The
  probe's outcome selects the arm shape; β-2 must not start until β-1's

- **R4 (G1 scope):** ship the `stack()`-of-bare-identifiers partial FIRST
  (covers the measured 2/10 Bakery cases via the existing
  `parseExpression(a.value, …)` per-arg path, pS:498-501). General nested
  reference is stretch and only attempted if the partial lands clean with
  time remaining. D-02 permits either.
- **R5 (D-03 column/pagination):** the exact `code_v1` body column and
  pagination are not determinable from local source — the maintainer
  script (task V-1) confirms them against upstream `website/src/repl/util.mjs`
  at the pinned SHA at run time.
- **R6 (G1 loc choice):** substituted binding subtrees keep their original
  RHS source offsets → click-to-source lands on the DEFINITION site. This
  is the documented v1 choice (matches issue #134's loc note).

## PRE-MORTEM (the single most-likely wrong-but-green outcome)

**A walker-migration that passes the 16-corpus snapshot + 1551 editor
tests but silently shifts offsets so runtime `loc`/highlighting breaks.**

Why this is the deadliest: snapshots assert IR *structure*, not absolute
`loc` offsets against the ORIGINAL source. A primitive that consumes the
right *tokens* but returns a wrong absolute index will produce identical
structure (green) while every Play node's `loc.start` drifts — and runtime
click-to-source / highlight-on-event is observation-only, NOT unit-testable
(prior project lesson: runtime loc fidelity is verified by observation, not
assertion).

**Designed-in detection (mandatory in α-3 and the verification wave):**
add a *loc-offset assertion harness* — for each of the 16 corpus tunes,
after parse, walk the IR and assert every leaf `loc.start` index, when
sliced out of the ORIGINAL source string, equals the expected token text
(e.g. the mini-notation string the Play came from). This converts the
observation-only property into a deterministic CI check FOR THE CORPUS.
Run it before AND after every walker reroute; the diff must be empty.
This is the regression oracle the snapshot alone cannot provide.
