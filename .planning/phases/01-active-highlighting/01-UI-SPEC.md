| Property | Value |

## Color

| Role | Dark Value | Light Value | Usage |
|------|-----------|-------------|-------|
| Dominant (60%) | `#090912` (`--background`) | `#f8f7ff` (`--background`) | Editor background, page background |
| Secondary (30%) | `#0f0f1a` (`--surface`) | `#ffffff` (`--surface`) | Toolbar background, panel surfaces, elevated cards |
| Accent (10%) | `#8b5cf6` (`--accent`) | `#7c3aed` (`--accent`) | See reserved list below |
| Destructive | `#f87171` (red-400 — inline, no token) | `#f87171` (same) | Error badge text and background tint in Toolbar |

Source: `packages/editor/src/theme/tokens.ts`

**Accent reserved for:**
1. `strudel-active-hap` decoration background — `rgba(var(--accent-rgb), 0.3)` via `--code-active-hap` token
2. `strudel-active-hap` decoration outline — `rgba(var(--accent-rgb), 0.5)` at 1px solid
3. `strudel-active-hap` decoration glow — `box-shadow: 0 0 8px rgba(var(--accent-rgb), 0.3)`
4. Play button background (active/playing state: `rgba(139,92,246,0.15)` bg + accent border)
5. Play button fill (idle state: `var(--accent)` solid background with white text)
6. Monaco cursor caret (`--code-caret`)
7. Monaco selection background (`--code-selection`)

**Per-note color override:** When `HapEvent.color` is a non-null CSS color string, the decoration background uses that color at 30% opacity in place of the accent token. The outline uses the same color at 50% opacity. This is injected as a dynamic CSS class (e.g., `strudel-active-hap--{sanitizedColor}`), cached in a `Map<string, string>` to avoid duplicate `<style>` injections. The class selector must be scoped to avoid collision with other editors on the same page.

## Decoration Visual Contract

This section is specific to Phase 1 and describes the exact visual appearance of the `strudel-active-hap` CSS class.

### Default state (accent color)

```css
.strudel-active-hap {
  background: rgba(var(--accent-rgb, 139, 92, 246), 0.3);
  border-radius: 2px;
  outline: 1px solid rgba(var(--accent-rgb, 139, 92, 246), 0.5);
  box-shadow: 0 0 8px rgba(var(--accent-rgb, 139, 92, 246), 0.3);
  /* NO transition — snap-on, snap-off per CONTEXT.md decision */
}
```

Source: Existing stub in `StrudelMonaco.tsx` lines 119–128, with `transition` removed per CONTEXT.md ("snap-off — no CSS fade transition").

### Per-note color variant (Claude's discretion)

When `HapEvent.color` is non-null, inject a class named `strudel-active-hap--{hex-or-safe-id}`:

```css
.strudel-active-hap--{safeId} {
  background: rgba({r}, {g}, {b}, 0.3);
  outline: 1px solid rgba({r}, {g}, {b}, 0.5);
  box-shadow: 0 0 8px rgba({r}, {g}, {b}, 0.3);
}
```

The `safeId` is a stable hash or base64 of the color string, ensuring valid CSS class names. Injected once per unique color via a singleton `<style>` tag appended to `document.head`, tracked in a module-level `Map<string, boolean>`.

### Monaco decoration options

```typescript
{
  className: 'strudel-active-hap',         // full-glyph background + outline
  stickiness: 1,                            // NeverGrowsWhenTypingAtEdges
  // inlineClassName NOT used — applies to text span only, not full glyph background
}
```

Source: RESEARCH.md verified against `monaco.d.ts` lines 1685–1694, 2818–2847.

### Timing behavior (not visual, but interaction contract)

| State | Trigger | Visual Result |
|-------|---------|---------------|
| Note scheduled | HapEvent emitted | No visual change yet |
| Note playing | `setTimeout` fires at `Math.max(0, scheduledAheadMs)` | Decoration appears (snap-on) |
| Note ended | `setTimeout` fires at `showDelay + audioDuration * 1000` | Decoration disappears (snap-off) |
| Engine stopped | `clearTimeout` on all pending IDs + `collection.clear()` | All decorations removed immediately |
| `evaluate()` called | Stale decorations cleared before new pattern starts | Editor resets to clean state |
| `loc` is null | Guard fires, skip decoration | No visual change |

## Accessibility Contract

| Concern | Decision |
|---------|----------|
| Decoration contrast | The `strudel-active-hap` background at 30% opacity over the dark editor background (`#090912`) results in an effective foreground of `rgba(139,92,246,0.3)` tinted over the code color — decorations are supplemental, not the primary reading surface. Code text remains `--code-foreground` at full opacity. |
| Keyboard users | Ctrl+Enter / Ctrl+. shortcuts already in place via Monaco `addAction`. No new keyboard affordances needed for highlighting. |
| Reduced motion | Per CONTEXT.md the decoration is snap-off (no transition). This is inherently reduced-motion safe — no CSS animation or transition is present. |
| Screen readers | Monaco editor text is not exposed via ARIA in a meaningful way; decorations are CSS-only and carry no ARIA semantics. Acceptable for an audio production tool where visual feedback is supplemental. |
