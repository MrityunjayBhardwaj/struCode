# Plan 19-01: IR Types + Backward-Compatible Migration

## Objective

Create `packages/editor/src/ir/` with the canonical IREvent and IRPattern types, pure transform functions, and backward-compatible re-exports so that all existing viz/engine code continues to work unchanged. NormalizedHap and PatternScheduler become thin aliases pointing at the new IR module.

## Context

- NormalizedHap (9 fields) is consumed by 13 files across viz sketches, BufferedScheduler, highlighting, and tests.
- PatternScheduler (now + query) lives in `visualizers/types.ts` and is implemented by StrudelEngine and BufferedScheduler.
- IREvent is NormalizedHap + 4 optional fields: `loc`, `trackId`, `type`, `params`. All new fields are optional, so IREvent is structurally assignable to NormalizedHap.
- BufferedScheduler uses audio-seconds in begin/end while Strudel uses cycle-time. This plan does NOT unify time domains -- it documents the split via IRV1 and keeps each engine's convention intact.
- Transform tree (IRNode, provenance) is deferred to Phase 20+. This plan ships flat events and pure utility functions only.

## Invariants

- **IRV1 (Time Domain Consistency):** begin/end must match the scheduler's now(). Each engine keeps its own time domain. No cross-engine time normalization in this phase.
- **IRV2 (Field Presence):** Producers should compute both note and freq when possible. Consumers must tolerate either being null.
- **IRV3 (Backward Compatibility):** IREvent must be structurally assignable to NormalizedHap. All new fields are optional. Existing import paths continue to resolve via re-exports.
- **UV6 (Observation Without Mutation):** IR types are read-only snapshots. Transform functions return new arrays/objects, never mutate input.

  <action>
    Create `packages/editor/src/ir/transforms.ts` with 4 pure functions:

    1. `merge(...arrays: IREvent[][]): IREvent[]`
       - Concatenate and sort by begin time.

    2. `transpose(events: IREvent[], semitones: number): IREvent[]`
       - Return new array. For each event:
         - If note is number: note + semitones, recompute freq (440 * 2^((newNote-69)/12))
         - If note is string: convert to MIDI via noteToMidi, add semitones, store as number, compute freq
         - If note is null: pass through unchanged
       - Import noteToMidi from ../engine/noteToMidi

    3. `timestretch(events: IREvent[], factor: number): IREvent[]`
       - Return new array. For each event: begin * factor, end * factor, endClipped * factor (if present).

    4. `filter(events: IREvent[], pred: (e: IREvent) => boolean): IREvent[]`
       - Thin wrapper: events.filter(pred). Exists for API completeness and future instrumentation.

    All functions: never mutate input, always return new arrays with spread copies of events where fields change.
  </action>
  <ownership>
    Transform functions: created by this task, consumed by future viz/DAW views (no consumer yet -- shipped for API completeness)
  </ownership>
  <lifecycle>
    1. Create file -- sync -- this task
    2. Pure functions, no lifecycle dependencies
  </lifecycle>
  <pre_mortem>
    Most likely error: Mutating input events instead of creating copies (violates UV6).
