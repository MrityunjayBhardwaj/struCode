title: Projects
description: Local-first workspaces with CRDT sync, auto-snapshots, and share links.

A Stave **project** is a collection of files, a layout, and a pinned backdrop. Everything persists to your browser's IndexedDB — no account, no server.

## File kinds

| Extension | Runtime |
|---|---|
| `.strudel` | Strudel pattern |
| `.sonicpi` | Sonic Pi pattern |
| `.p5` | p5.js sketch (also usable as a named viz) |
| `.hydra` | Hydra chain (also usable as a named viz) |
| `.md` | Markdown — notes |

Pattern files (`.strudel`, `.sonicpi`) drive audio. Viz files (`.p5`, `.hydra`) can be opened as their own tab **and** referenced by name from `.viz("…")` calls inside patterns.

## Sync model

Every project is a Yjs document. Changes propagate to other open tabs instantly over a BroadcastChannel. Disconnect, edit offline, reconnect — the CRDT merges cleanly.

## Snapshots

- **Auto** — every 60 seconds of idle, Stave captures a named snapshot (`auto-14:32`). The last 10 are retained.
- **Manual** — save a named snapshot from the Version History panel. Manual snapshots are never pruned.
- **Restore** — any snapshot can be restored non-destructively (a new auto-snapshot captures the pre-restore state first).

## Share links

File menu → **Copy Share Link**. Produces a fragment URL like `https://stave.live/#share=eJxb…` containing the compressed project. Paste into a chat. The recipient visits the URL, sees an "Import shared project?" prompt.

Share URLs are purely client-side — they contain the full project data, they don't point at anything on a server.

## Import / export

- **Export** → `.zip` with every file, preserving the directory structure.
- **Import** → drop a `.zip` on the sidebar or use File → Import.
