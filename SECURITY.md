# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in Stave, please report it responsibly.

**Do not open a public issue for security vulnerabilities.**

Instead, please report vulnerabilities via [GitHub's private vulnerability reporting](https://github.com/MrityunjayBhardwaj/stave/security/advisories/new).

Alternatively, email the maintainer directly. You can find contact information on the [GitHub profile](https://github.com/MrityunjayBhardwaj).

## What to Include

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

## Response Timeline

- **Acknowledgment:** within 48 hours
- **Initial assessment:** within 7 days
- **Fix or mitigation:** as soon as practical, depending on severity

## Scope

This policy covers the Stave codebase and its official deployment. Third-party dependencies have their own security policies.

## Supported Versions

Only the latest version on the `main` branch is actively maintained.
